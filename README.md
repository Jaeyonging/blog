# Jaeyonging 블로그

## 기능
- 블로그 포스팅 및 관리
- 관리 패널을 통한 콘텐츠 관리
- 로그인 및 사용자 정보에 대한 외부 API 통합
- Tailwind CSS를 사용한 반응형 디자인
- Quill로 강화된 리치 텍스트 편집
- 개인 서버 환경에 맞춰 자동화된 관리 시스템

### 배포

Apache2 웹 서버를 사용하여 개인 서버에서 배포 하였으며, 도메인은 [jaeyonging.com](http://jaeyonging.com) 입니다.

### 환경 변수

이 프로젝트는 특정 환경 변수에 의존합니다:

- `VITE_API_URL`: API 백엔드 서버의 URL.

프로젝트의 루트 디렉토리에 `.env` 파일을 만들어 변수들을 설정하세요.

## 배우고 느낀 점

이 프로젝트를 통해 저는 여러 가지 중요한 기술과 개념을 배우고 경험하게 되었습니다.

1. **상태 관리의 중요성**: `Zustand`를 사용하여 전역 상태를 효과적으로 관리할 수 있었습니다. 이를 통해 페이지가 새로고침되더라도 데이터가 유지되도록 `persist` 미들웨어를 적용하여 사용자 경험을 향상시켰습니다.

2. **자동화와 관리의 효율성**: Admin 패널 및 자동화된 관리 시스템 구축을 통해 어떻게 백엔드와 프론트엔드를 조화롭게 통합할 수 있는지 배웠습니다. 특히, 개인 서버 환경에서의 자동화는 유지보수와 운영의 효율성을 높이는 데 큰 도움이 되었습니다.

3. **Apache2 서버 설정 경험**: 개인 서버에 Apache2를 이용하여 직접 배포함으로써 실제 서버 환경에서의 문제 해결 능력과 서버 관리 역량을 키울 수 있었습니다. 이를 통해 더 나은 사용자 경험을 제공하는 방법을 배우게 되었습니다.

4. **보안과 접근 제어**: `ProtectedRoute`와 같은 컴포넌트를 구현함으로써, 민감한 정보가 포함된 페이지에 대한 접근을 제어하는 방법을 익혔습니다. 이는 보안성을 높이는 데 기여했습니다.
